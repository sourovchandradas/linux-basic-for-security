1. Introduction to Environment Variables
------------------------------------------------------------------------
* What are Environment Variables?
   -> Environment variables are process-wide variables built into your system.
   -> They control the way your system looks, acts, and "feels" to the user.
   -> They are inherited by any child shells or processes.
   -> Environment variables are always uppercase (e.g., HOME, PATH, SHELL).

* Shell Variables vs Environment Variables:
   -> Environment Variables  : Process-wide, inherited by child processes, uppercase.
   -> Shell Variables        : Only valid in the shell they are set in, typically lowercase.

* Why Manage Environment Variables?
   -> To optimize system performance.
   -> To tailor your work environment to your needs.
   -> To customize appearance and behavior.
   -> To cover your tracks (e.g., disabling command history).
   -> To configure tools and applications efficiently.

* Variable Format:
   -> Single value: KEY=value
   -> Multiple values: KEY=value1:value2:value3
   -> Values with spaces: KEY="value with spaces" (use quotation marks)


2. Viewing Environment Variables
------------------------------------------------------------------------
* The env Command (View Default Environment Variables):
   kali > env
   
   Output Explanation:
   XDG_VTNR=7
   TERM=xterm-256color
   SHELL=/bin/bash
   USER=root
   PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/sbin:/bin
   HOME=/root
   
   -> Displays all default environment variables set by the system.
   -> Shows uppercase variable names and their values.
   -> These variables come pre-configured on every Linux system.

* Important Default Variables:
   HOME    = User's home directory (e.g., /root for root user)
   PATH    = Directories where the system searches for executable commands
   SHELL   = Current shell being used (e.g., /bin/bash)
   USER    = Username of the current user
   TERM    = Terminal type (e.g., xterm-256color)
   LANG    = System language and locale settings
   PWD     = Present/Current working directory


3. Viewing All Variables (Including User-Defined)
------------------------------------------------------------------------
* The set Command (View All Variables):
   kali > set
   
   -> Displays ALL variables: environment variables, shell variables, and functions.
   -> Output is very long and may not fit on one screen.
   -> Shows user-defined variables and command aliases as well.

* Making Output Manageable with more:
   kali > set | more
   
   BASH=/bin/bash
   BASHOPTS=checkwinsize:cmdlist:complete_fullquote:expand_aliases:extglob
   BASH_ALIASES=()
   --snip--
   
   -> Piping to 'more' makes output scroll line by line.
   -> Press Enter to advance to next line.
   -> Press Q to quit and return to prompt.

* Using grep to Filter Specific Variables:
   kali > set | grep HISTSIZE
   HISTSIZE=1000
   
   -> Searches for a specific variable without viewing all variables.
   -> Much more convenient when looking for one particular variable.
   -> Example: Finding HISTSIZE, which stores command history count.


4. Understanding Key Environment Variables
------------------------------------------------------------------------
* HISTSIZE (Command History Size):
   -> Controls maximum number of commands stored in command history.
   -> Default value: 1000 (stores last 1,000 commands).
   -> Accessible via up/down arrow keys in terminal.
   -> Set to 0 to disable history (for stealth/security).
   
   Example:
   kali > set | grep HISTSIZE
   HISTSIZE=1000

* PATH (Command Search Path):
   -> Most important variable for system functioning.
   -> Contains directories separated by colons (:).
   -> System searches these directories when you type a command.
   -> Example: /usr/local/sbin:/usr/local/bin:/usr/sbin:/sbin:/bin
   
   Explanation:
   -> If you type 'ls', system searches PATH directories for 'ls' executable.
   -> First match is executed; order matters.
   -> Adding custom directories to PATH allows running scripts from anywhere.

* PS1 (Prompt String):
   -> Controls what your command prompt looks like.
   -> Default might be: $ (regular user) or # (root user).
   -> Can be customized with colors, username, hostname, directory info.
   
   Example:
   kali > echo $PS1
   \u@\h:\w$
   (\u=username, \h=hostname, \w=working directory)

* HOME (Home Directory):
   -> Absolute path to current user's home directory.
   -> For root: /root
   -> For regular user: /home/username
   -> Used by many applications to store config files.

* SHELL (Default Shell):
   -> Path to the default shell being used.
   -> Usually: /bin/bash (Bash shell)
   -> Can be changed for different shell preferences.


5. Changing Variable Values for Current Session
------------------------------------------------------------------------
* Syntax:
   VARIABLE_NAME=new_value

* Example-01: Disable Command History
   kali > HISTSIZE=0
   
   Effect:
   -> Command history is no longer stored.
   -> Up/down arrow keys won't recall previous commands.
   -> Useful for security/stealth (no evidence of activity).

* Example-02: Modify Command Prompt
   kali > PS1="[HACKED] $ "
   
   Output:
   [HACKED] $
   
   Effect:
   -> Prompt changes immediately to show "[HACKED]".
   -> Only affects current session.
   -> Reverts to default when terminal closes.

* Example-03: Add Directory to PATH
   kali > PATH=$PATH:/home/user/scripts
   
   Effect:
   -> Scripts in /home/user/scripts can now be executed directly.
   -> Colon (:) separates directories in PATH.
   -> $PATH preserves existing directories; new one is appended.

* Important Note:
   ⚠️ Changes only apply to CURRENT SESSION.
   -> Closing terminal = changes are lost.
   -> Values reset to defaults on next login.


6. Making Variable Changes Permanent
------------------------------------------------------------------------
* The export Command:
   -> Exports variable from current shell to all child processes.
   -> Makes changes persist in current session and new child shells.
   -> Syntax: export VARIABLE_NAME=value
   
   OR
   
   -> Export after changing: export VARIABLE_NAME

* Example-01: Permanent HISTSIZE Change (Session-wide)
   kali > HISTSIZE=0
   kali > export HISTSIZE
   
   Effect:
   -> All new child shells inherit HISTSIZE=0.
   -> But still lost when terminal closes.

* Example-02: Undo Variable Change
   kali > HISTSIZE=1000
   kali > export HISTSIZE
   
   -> Resets HISTSIZE back to default value.
   -> Useful if you want to restore original behavior.

* Making Changes Survive Terminal Restart:
   -> Edit shell configuration files: ~/.bashrc or ~/.profile
   -> Add lines like: export HISTSIZE=0
   -> Changes apply automatically on next login.

* Backup Before Modifying:
   kali > echo $HISTSIZE > ~/valueofHISTSIZE.txt
   kali > set > ~/valueofALL.txt
   
   Effect:
   -> Saves current values before modification.
   -> Allows reverting changes if something breaks.
   -> Good practice when experimenting with variables.


7. Viewing Variable Values
------------------------------------------------------------------------
* Using echo to Display a Variable:
   kali > echo $HISTSIZE
   1000
   
   -> $ prefix is required to expand variable.
   -> Without $, it just prints the literal text "HISTSIZE".

* Example-01: View HOME Directory
   kali > echo $HOME
   /root

* Example-02: View PATH
   kali > echo $PATH
   /usr/local/sbin:/usr/local/bin:/usr/sbin:/sbin:/bin

* Example-03: View Multiple Variables
   kali > echo "User: $USER, Home: $HOME, Shell: $SHELL"
   User: root, Home: /root, Shell: /bin/bash


8. Practical Scenarios - When to Modify Variables
------------------------------------------------------------------------
* Scenario-01: Hide Command History (Stealth)
   kali > HISTSIZE=0
   kali > export HISTSIZE
   
   Why:
   -> Hacker/pentester doesn't want to leave evidence of commands used.
   -> System won't store executed commands.

* Scenario-02: Add Custom Tools to PATH
   kali > PATH=$PATH:/opt/mytools
   kali > export PATH
   
   Why:
   -> Avoids typing full path every time you run a custom tool.
   -> Makes tools accessible from anywhere in filesystem.

* Scenario-03: Customize Prompt (Visual Identification)
   kali > export PS1="[ROOT@KALI] \w $ "
   
   Why:
   -> Helps identify which user/host you're on.
   -> Useful when managing multiple systems.

* Scenario-04: Temporary Variable for Script/Tool Configuration
   kali > export JAVA_HOME=/usr/lib/jvm/java-11
   kali > export PATH=$PATH:$JAVA_HOME/bin
   
   Why:
   -> Tools need specific configuration to function.
   -> Java tools, in particular, require JAVA_HOME.

* Scenario-05: Reset Variable to Default
   kali > HISTSIZE=1000
   kali > export HISTSIZE
   
   Why:
   -> Restore original behavior after testing.


9. Important Rules and Best Practices
------------------------------------------------------------------------
⚠️ Key Rules to Remember:

   1. Environment Variable Naming:
      -> Always UPPERCASE (HOME, PATH, SHELL, USER, etc.).
      -> Shell variables are typically lowercase.

   2. Setting vs. Exporting:
      -> Setting (HISTSIZE=0): Only current shell.
      -> Exporting (export HISTSIZE): Current + child shells.
      -> Neither makes changes permanent across sessions.

   3. Making Changes Permanent:
      -> Edit ~/.bashrc or ~/.profile.
      -> Add: export VARIABLE_NAME=value
      -> Reload: source ~/.bashrc

   4. Accessing Variable Values:
      -> Use $ prefix: echo $HOME (correct).
      -> No $ prefix: echo HOME (just prints "HOME").

   5. PATH Modifications:
      -> Always preserve existing PATH: $PATH:/new/path.
      -> Never replace: PATH=/new/path (will break system).

   6. Multiple Values in Variables:
      -> Separator: Colon (:)
      -> Example: PATH=dir1:dir2:dir3

   7. Spaces in Values:
      -> Use quotation marks: PS1="My Custom Prompt"

   8. Backup Before Modifying:
      -> Save current values: set > ~/backup.txt
      -> Allows reverting if something breaks.

   9. Security/Stealth Implications:
      -> HISTSIZE=0 hides command history.
      -> Environment variables can reveal system info.
      -> Hackers may modify PS1 to disguise their presence.

* One-liner to Remember:
   -> env = view environment variables.
   -> set = view all variables.
   -> export = make changes persist to child processes.
   -> Edit ~/.bashrc = make changes permanent across sessions.
