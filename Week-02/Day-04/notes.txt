1. Adding Repositories to Your 'sources.list' File
------------------------------------------------------------------------
* Repositories are servers that host software for specific Linux distributions.
* The 'sources.list' file defines where the system searches for software. 
  It is located at: /etc/apt/sources.list
* Kali specializes in security tools, so adding backup repositories (like Ubuntu 
  repositories) ensures you can find everyday or specialty software.

Editing the file:
kali > mousepad /etc/apt/sources.list

Debian/Kali Repository Categories:
* main        : Supported open-source software.
* universe    : Community-maintained open-source software.
* multiverse  : Software restricted by copyright or legal issues.
* restricted  : Proprietary device drivers.
* backports   : Packages from later software releases.

⚠️ Warning: Avoid using 'testing', 'experimental', or 'unstable' repositories 
            as untested software might break your system.

Example: Adding third-party PPA locations for Oracle Java 8:
deb http://ppa.launchpad.net/webupd8team/java/ubuntu trusty main
deb-src http://ppa.launchpad.net/webupd8team/java/ubuntu precise main


2. Using a GUI-based Installer
------------------------------------------------------------------------
Newer versions of Kali Linux do not include a graphical package manager by 
default, but you can manually install tools like 'Synaptic' or 'Gdebi'.

Installation Example (Synaptic):
kali > sudo apt-get install synaptic

How to use it:
1. Open it from the terminal by typing 'synaptic' or via Settings -> Synaptic Package Manager.
2. Click 'Search' and enter the package name (e.g., snort).
3. Mark the checkbox next to the package and click 'Apply' to install it 
   along with its dependencies.


3. Installing Software with 'git'
------------------------------------------------------------------------
When a tool is brand new or not hosted in any official repository, it can 
often be found on GitHub. You can download and install it directly from the 
terminal using the 'git clone' command followed by the repository URL.

Example: Cloning 'bluediving' (a Bluetooth hacking suite):
kali > git clone https://www.github.com/balle/bluediving.git

Output Verification:
* 'git clone' copies all project files and creates a new directory named after the tool.
* Verify the download using the long listing command:
  kali > ls -l

Example Output:
drwxr-xr-x 7 root root  4096 Jan 10 22:19 bluediving
drwxr-xr-x 2 root root  4096 Dec  5 11:17 Desktop
