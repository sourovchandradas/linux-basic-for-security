========================================================================
                    ADDING AND REMOVING SOFTWARE
========================================================================

1. Introduction to Software Management
------------------------------------------------------------------------
Managing software (adding/removing packages) is a fundamental OS task. 
Linux software often relies on "dependencies" or "libraries" (other software 
required to run successfully). A software package bundles these files together 
along with installation scripts.

Key installation methods covered:
* apt package manager (Advanced Packaging Tool)
* GUI-based installation managers
* git


2. Using 'apt' / 'apt-get' to Handle Software
------------------------------------------------------------------------
In Debian-based distributions (like Kali and Ubuntu), 'apt' is the default 
package manager. While 'apt' is popular for simplicity, 'apt-get' offers 
more advanced functionality and is essential to learn early on.

IMPORTANT: Most apt commands require root privileges and should be run with 'sudo'

A. Searching for a Package
Before downloading, check if the package exists in your system's repository cache.
Syntax:
apt-cache search keyword

Example (Searching for the Snort IDS):
kali > apt-cache search snort

B. Adding Software (Installation)
Downloads and installs the specified package from the repository.
Syntax:
sudo apt-get install packagename

Example:
kali > sudo apt-get install snort
(Type 'Y' or 'y' when prompted "Install these packages without verification?")


3. Removing Software (Uninstall & Purge)
------------------------------------------------------------------------
* Standard Removal: Removes the software but retains configuration files 
  (allows future reinstallation without reconfiguring).
  kali > sudo apt-get remove snort

* Purging (Complete Removal): Removes the software along with all its 
  associated configuration files.
  kali > sudo apt-get purge snort

* Cleaning Dependencies (autoremove): When software is uninstalled, its 
  isolated dependencies/libraries remain behind. Use 'autoremove' to clean 
  them up and free disk space.
  kali > sudo apt autoremove snort


4. Updating vs. Upgrading Packages
------------------------------------------------------------------------
There is a fundamental difference between updating and upgrading in Linux:
* Update  : Refreshes the local database list of available packages from the repository.
* Upgrade : Downloads and installs the actual latest versions of the software.

A. Updating Packages
Command:
kali > sudo apt-get update

Note: Look for "Reading package lists... Done" to confirm a successful update.

B. Upgrading Packages
Requires root privileges. It checks all repository-linked tools on your system 
and upgrades them simultaneously. This process can be highly time-consuming.
Command:
kali > sudo apt-get upgrade

Note: Review the estimated disk space usage in the terminal before typing 
'Y' to proceed with the system-wide upgrade.


5. Common Errors & Troubleshooting
------------------------------------------------------------------------
* "Permission Denied" - You forgot 'sudo'. Always use 'sudo' for install/remove/upgrade
* "E: Unable to locate package" - Run 'sudo apt-get update' first to refresh package list
* "E: Could not open lock file" - Another apt process is running. Wait and try again
* "Broken packages" - Run 'sudo apt --fix-broken install' to fix dependency issues


6. Practice Exercises
------------------------------------------------------------------------
Try these commands to practice:
1. Search for a package: apt-cache search curl
2. Install a package: sudo apt-get install curl
3. Check if installed: which curl
4. Remove a package: sudo apt-get remove curl
5. Update package list: sudo apt-get update
6. List installed packages: apt list --installed
